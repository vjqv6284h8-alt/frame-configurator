# Frame Configurator

Локальное веб-приложение для расчета рам, профилей, узлов и фурнитуры с хранением проекта в файле на сервере.

## Что это за сайт

Приложение помогает:

- конфигурировать раму и сегменты (FIX/SL),
- вести базы профилей, фурнитуры, готовой фурнитуры, готовых профилей и узлов,
- назначать системные узлы из базы,
- получать спецификацию,
- экспортировать результат в Excel/PDF.

Проект автоматически сохраняется на сервер в `server-data/project.json`.

## Структура сайта (вкладки)

1. `Конфигуратор`
   - параметры проекта (номер, составил),
   - размеры рамы,
   - серия профиля, тип нижнего профиля створки (включая FreeWay),
   - сегменты и направление открывания,
   - расчет и таблица спецификации.

2. `База профилей`
   - CRUD профилей (артикул, название, длина),
   - загрузка/просмотр сечения профиля.

3. `База фурнитуры`
   - CRUD штучной фурнитуры (ручки, крепеж, каретки и т.д.).

4. `Готовая фурнитура`
   - CRUD готовых наборов фурнитуры (например, комплект замков/кареток),
   - используется как отдельный тип позиции в составе готового профиля.

5. `Готовые профили`
   - CRUD сборок (артикул, название, сечение),
   - конструктор состава сборки: в строке можно выбрать профиль, фурнитуру или готовую фурнитуру,
   - поиск в поле выбора по артикулу/названию.

6. `База узлов`
   - CRUD шаблонов узлов,
   - CRUD пользовательских узлов с формулами и режимами количества,
   - использование профилей/фурнитуры в составе.

7. `Настройки`
   - назначение системных узлов/сборок для расчета,
   - параметры вычетов для серий,
   - ручная синхронизация с сервером (при необходимости),
   - сброс проекта к шаблону.

## Архитектура

- **Frontend:** React + TypeScript + Vite (`src/App.tsx`).
- **Backend:** Express (`server.js`).
- **Хранилище проекта:** JSON-файл `server-data/project.json`.
- **Сборка для стабильного режима:** `dist/` раздается Express-сервером.

## Хранение данных

- Источник правды: `server-data/project.json`.
- Формат файла:
  - `updatedAt` — время последнего сохранения,
  - `data` — объект проекта (все вкладки и настройки).
- При старте UI загружает проект с сервера.
- Все изменения автосохраняются на сервер.
- Активная вкладка сохраняется в URL hash (`#config`, `#assemblies` и т.д.), поэтому после обновления страницы остается текущая вкладка.

## API

`server.js` предоставляет:

- `GET /api/health` — проверка сервера.
- `GET /api/project` — получить сохраненный проект.
- `PUT /api/project` — сохранить проект.

## Запуск

```bash
npm install
```

### Dev (только фронтенд)

```bash
npm run dev
```

### Dev по LAN

```bash
npm run dev:lan
```

### Stable по LAN (рекомендуется для работы в сети)

```bash
npm run stable:lan
```

Команда:

- соберет `dist`,
- поднимет Express-сервер на порту `5181`,
- откроет сайт по локальному и LAN-адресу текущей машины.

## Полезные скрипты

- `npm run build` — production-сборка.
- `npm run lint` — проверка линтером.
- `npm run preview` — предпросмотр Vite-сборки.
- `npm run server` — запуск Express-сервера.

## Основные возможности

- Расчет профилей по геометрии рамы и сегментов.
- Учет штучных позиций (фурнитура/готовая фурнитура).
- Работа с системными шаблонами узлов.
- Назначение узлов для разных режимов (включая FreeWay).
- Экспорт спецификации в Excel и PDF.
- Работа в локальной сети с общей серверной базой.

## AI review для Pull Request

В проект добавлен GitHub Action `.github/workflows/ai-pr-review.yml`, который:

- запускается на событиях PR (`opened/synchronize/reopened/ready_for_review`),
- берет diff между веткой PR и base-веткой,
- отправляет diff в LLM,
- публикует (или обновляет) комментарий `AI Review` в PR.

### Настройка

1. В GitHub репозитории откройте `Settings -> Secrets and variables -> Actions`.
2. Создайте secret `OPENAI_API_KEY`.
3. (Опционально) создайте variable `OPENAI_MODEL` (по умолчанию `gpt-4.1-mini`).
4. Убедитесь, что у workflow есть permission `pull-requests: write` (уже задано в файле).
